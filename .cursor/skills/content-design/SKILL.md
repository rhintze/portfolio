# Content Design

## Principles

1. **Clear over clever.** Every sentence should be immediately understood on
   first read. No jargon unless the audience expects it (recruiters, design
   leads, hiring managers).

2. **Concise.** Cut filler words. If a sentence works without a word, remove
   the word. Aim for 12–18 words per sentence max.

3. **Value-driven.** Lead with outcomes and impact, not activities.
   Say "Increased B2C transactions 2.6×" before "Led 70+ user interviews."

4. **Actionable.** CTAs tell the user exactly what happens next.
   "Read case study" > "Learn more". "View projects" > "See here".

5. **Consistent voice.** Professional but human. Confident, not boastful.
   First person sparingly. Let the work speak.

## Voice & Tone

| Attribute | Do | Don't |
|-----------|-----|-------|
| Confident | "I led design for…" | "I was lucky to work on…" |
| Specific | "70+ user interviews" | "extensive research" |
| Direct | "SaaS products from concept to launch" | "Various software products" |
| Human | "Helping solopreneurs build businesses" | "Enabling SMB digital transformation" |

### Case Studies Index

| Element | Guidelines |
|---------|------------|
| Page heading | "Case Studies" — plain, scannable |
| Page intro | One line, under 12 words. Set expectation. |
| Card title | Outcome-driven headline, ≤8 words |
| Card excerpt | 1–2 sentences, focus on approach + method |
| Card CTA | "Read case study →" — explicit action |
| Metric | Single hero number + short label (≤3 words) |

### Case Study Detail

| Element | Guidelines |
|---------|------------|
| Opening quote | Real user quote from research. Keep under 10 words. |
| Section titles | Uppercase, action-oriented: "My Role", not "About My Role" |
| Body paragraphs | 2–3 short paragraphs per section. Lead with the "so what." |
| Bullet lists | Start each item with a strong verb |
| Metric labels | Describe the impact, not the metric name |
| Image descriptions | Describe what the user sees + the design intent |

## Image Alt Text

- Describe the **visual content**, not the design decision.
- Include context: "Mobile onboarding screen showing business name input field"
- For decorative images, use `alt=""`.
- For portfolio screenshots, describe the screen state, not "screenshot of".

## Review Checklist

When reviewing copy on any page, verify:

- [ ] Every heading passes the "scan test" — skimming headings alone tells the story
- [ ] No sentence exceeds 20 words
- [ ] Numbers and metrics appear before their explanation
- [ ] CTAs use action verbs
- [ ] No passive voice
- [ ] No filler: "very", "really", "just", "actually", "in order to"
- [ ] Alt text on every meaningful image
- [ ] Meta title ≤60 chars, meta description ≤155 chars
